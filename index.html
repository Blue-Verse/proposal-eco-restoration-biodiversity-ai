프로젝트 소개

핵심 요구사항 분석

  • AI 모델 고도화 — CAPTAIN 알고리즘 소스 리뷰·구조 분석, 신규 환경 변수 추가, 예측 정확도·연산 속도 개선, 시뮬레이션 출력 포맷 최적화

  • 동적 시각화 — 시뮬레이션 결과값에 따른 시간 흐름별 동적 그래프 시각화
  • 실시간 데이터 연동 — 외부 토양·환경 DB 실시간 연동으로 최신 데이터 반영
  • 성능 개선 — 좌표 변환·데이터 조회 처리 속도 단축
  • 통합 연동 — 기존 파이썬 기반 웹 서비스의 프론트엔드·백엔드 통합 연동

제안 포인트

  • 운영 서비스 고도화에 최적화된 접근 — 신규 개발이 아닌 운영 중 서비스인 만큼, 먼저 CAPTAIN 알고리즘과 현행 파이프라인(주소→좌표변환→경계설정→토양조회→시뮬레이션→시각화)을 정밀 분석해 리스크를 식별하고 단계적·무중단 지향으로 개선합니다.
  • Python 기반 데이터·AI 역량 — Python 데이터 처리·모델 연동 프로젝트 다수 수행 경험으로, 모델 구조 분석·신규 변수 반영·연산 최적화(벡터화·캐싱·병렬)를 안정적으로 진행합니다.
  • 데이터 기반 동적 시각화 경험 — 데이터 대시보드·실시간 시각화 구현 경험을 바탕으로 시간 흐름별 동적 그래프와 지도 폴리곤 편집 UX를 구현합니다.
  • 외부 연동·성능 설계 — 다수의 외부 API/DB 실시간 연동 및 좌표·데이터 조회 성능 개선 경험으로 응답 속도를 단축합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

생태 복원 대상지의 환경 데이터를 활용해 생물다양성을 예측하는 솔루션이 CAPTAIN 논문·공개 알고리즘 기반으로 파이썬 환경에서 운영 중입니다. 현재 주소 입력 → 좌표 변환 → 대상지 경계 설정 → 토양 데이터 조회 → 시뮬레이션 실행 → 결과 이미지·그래프 시각화로 이어지는 기본 파이프라인이 구축되어 있습니다. 본 프로젝트의 목적은 (1) 기존 모델에 신규 환경 변수를 추가하여 예측 성능을 높이고, (2) 사용자가 웹에서 직접 경계를 설정하거나 실시간 데이터를 반영해 동적 시각화 결과를 확인할 수 있도록 웹 서비스의 편의성·기능성을 고도화하는 것입니다.

2. 사용자 정의

사용자 유형설명주요 기능
분석/이용자복원 대상지 예측을 수행하는 실무 사용자주소 입력, 지도 폴리곤 경계 설정·수정, 시뮬레이션 실행, 동적 시각화 결과 확인
운영 관리자모델·데이터 운영 담당외부 DB 연동 상태 확인, 모델 변수·파라미터 관리, 결과 데이터 관리

3. 기능 요구사항

3.1 AI 예측 모델 고도화

  • [필수] CAPTAIN 알고리즘 소스 리뷰·구조 분석 및 문서화
  • [필수] 신규 환경 변수 추가 및 모델 재학습 파이프라인 구성
  • [필수] 예측 정확도 개선 (피처 엔지니어링, 검증 지표 정의)
  • [필수] 연산 속도 개선 (벡터 연산, 캐싱, 병렬 처리)
  • [필수] 시뮬레이션 결과 데이터 출력 포맷 최적화

3.2 웹 서비스 및 운영 기능 고도화

  • [필수] 지도상 대상지 폴리곤 경계 설정·수정 기능
  • [필수] 시뮬레이션 결과값 기반 시간 흐름별 동적 그래프 시각화
  • [필수] 외부 토양·환경 DB 실시간 연동을 통한 최신 데이터 반영
  • [필수] 좌표 변환·데이터 조회 처리 속도 단축
  • [필수] 기존 파이썬 기반 웹 서비스 FE/BE 통합 연동
  • [선택] 시뮬레이션 결과 이력 관리·비교

4. 비기능 요구사항

항목요구사항
성능좌표 변환·데이터 조회 응답 시간 단축, 시뮬레이션 비동기 처리로 UI 블로킹 방지
정확도신규 변수 반영 후 기존 대비 예측 지표(검증셋 기준) 개선 확인
안정성외부 DB 장애 시 캐시·폴백 처리, 운영 중 무중단 지향 배포
유지보수성모듈화·문서화된 소스 코드 및 알고리즘 분석 문서 이관
호환성기존 파이썬 스택과의 통합, 표준 좌표계(EPSG) 지원

5. 기술 제약사항

  • 기존 CAPTAIN 논문·공개 알고리즘 기반 파이썬 환경 유지
  • 외부 토양·환경 DB API의 가용성·요청 쿼터에 의존
  • 운영 중 서비스 — 점진적·무중단 배포 고려 필요

6. 범위 외 사항 (Out of Scope)

  • 신규 도메인 데이터 수집·라벨링 (제공 데이터 기준 진행)
  • 모바일 네이티브 앱 개발
  • 외부 유료 API/데이터 사용료
  • 하자 보수 기간 이후의 별도 유지보수
